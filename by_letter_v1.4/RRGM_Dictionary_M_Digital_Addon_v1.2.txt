MD001 model-router

Common meaning:
A component that selects which model or reasoning path receives an input.

Invariant MI [INV]:
Model-router preserves task distinction by directing each input toward the structure best suited to process it.

Active domains:
[DIG]

Domain entries:
[DIG] Routing uses declared signals, capacity, and failure modes rather than treating all models as interchangeable.

Count condition:
Counts when the component makes an explicit model-selection decision for an input.

Specific transfer boundary:
Routing should not silently strip refusal or provenance.

Epistemic status:
[DEF]
