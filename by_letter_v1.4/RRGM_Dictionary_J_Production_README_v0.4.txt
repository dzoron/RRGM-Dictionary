Production decisions
- J was the next-smallest source letter and adds a difficult judgment, justice, joke, join, joint, and jailbreak stress test.
- Every original source entry is preserved with its common meaning, example, and RRGM bridge.
- Each enhanced entry begins with a headword-specific invariant MI.
- Domain sets are assigned per entry rather than inherited from neighbouring words.
- One precise domain binding is preferred over decorative multi-domain coverage.
- Additional domains appear only where a bespoke implementation was written.
- Duplicate source headwords remain under their original IDs and are role-separated rather than cosmetically renamed.
- Count conditions are bound to the individual term or compound relation.
- Entry-level transfer boundaries appear only where a concrete confusion path was identified.
- Proof remains local to the domain where it was established.

Addon decision
- Mythic was selected because the source base contains many words capable of mythic implementation but lacks substantial mythic vocabulary.
- Biological was selected because the base contains comparatively thin life-science coverage under J and a useful set of missing terms could be defined without forcing the existing word list.
- Each addon contains 24 new entries absent from the source base.

Interpretation
The automated audit is a review gate, not proof of semantic perfection. Its purpose is to expose duplicated prose, collapsed neighbours, and generic domain templates before human or model review. The headword-swap test remains the controlling semantic test.
