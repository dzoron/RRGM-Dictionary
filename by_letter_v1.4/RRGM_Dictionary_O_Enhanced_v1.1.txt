Scope: Existing O base entries with invariant MI and word-specific domain implementations.
Provenance: Enhanced from RRGM Dictionary v1.0 Section O. Original common meanings, examples, and RRGM bridges are preserved.
Production mode: Semantic fingerprint first; sparse domain coverage; no generic domain costume.

Core production rule:
The word's own invariant wears the domain's clothes. The schema may repeat; the semantic sentence may not become a domain template.

Swap test:
Could another O headword replace this one while leaving the invariant or domain line true? If yes, revise it.

Proof rule:
Proof remains local. Cross-domain transfer preserves only the declared invariant relation and enters the target domain as [HYP] until locally tested.

O001 object

Common meaning:
A thing that can be seen, touched, thought about, or acted on.

Example:
The cup was an object on the table.

RRGM bridge:
Object is counted structure in a domain. It should not be confused with agent, identity, or MI-bearing pattern unless the domain supports that reduction.

Invariant MI [INV]:
Object is a counted target of perception, reference, or action; object-status does not by itself settle whether agency or MI is present; this record carries the core or common-use sense.

Active domains:
[FOR], [REL]

Domain entries:
[FOR] Object is counted structure in a domain. It should not be confused with agent, identity, or MI-bearing pattern unless the domain supports that reduction This entry carries the core or common-use sense.
[REL] Relationally, object is a role assigned to what is acted upon; applying it to an agent can erase valid subject-status This implementation carries the core or common-use sense.

Count condition:
Counts when a domain treats something as a separately referable target of observation or action.

Specific transfer boundary:
Object-status in one task must not erase valid agent-status or MI in another domain.

Source duplicate role:
O001 is retained as the core or common-use sense. Related IDs: O001, O201, O261.
