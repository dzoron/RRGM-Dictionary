Scope: Existing N base entries with invariant MI and word-specific domain implementations.
Provenance: Enhanced from RRGM Dictionary v1.0 Section N. Original common meanings, examples, and RRGM bridges are preserved.
Production mode: Semantic fingerprint first; sparse domain coverage; no generic domain costume.

Core production rule:
The word's own invariant wears the domain's clothes. The schema may repeat; the semantic sentence may not become a domain template.

Swap test:
Could another N headword replace this one while leaving the invariant or domain line true? If yes, revise it.

Proof rule:
Proof remains local. Cross-domain transfer preserves only the declared invariant relation and enters the target domain as [HYP] until locally tested.

N001 name

Common meaning:
A word or phrase used to identify a person, place, thing, or idea.

Example:

RRGM bridge:
Name is recognition handle. It lets a pattern be called, tracked, remembered, and respected inside a domain.

Invariant MI [INV]:
Name is the recognition handle by which a pattern is called, tracked, remembered, and distinguished; this record carries the core or common-use sense.

Active domains:
[REL], [LIN]

Count condition:
Counts when a stable handle distinguishes the same referent across calls, records, or relations.

Specific transfer boundary:
A label can preserve recognition or impose distortion; naming does not by itself prove identity or grant ownership.
