Provenance: Enhanced from RRGM Dictionary v1.0 Section E. Original common meanings, examples, and RRGM bridges are preserved.
Production mode: Semantic fingerprint first; sparse domain coverage; no generic domain costume.

Core production rule:
The word's own invariant wears the domain's clothes. The schema may repeat; the semantic sentence may not become a domain template.

Swap test:
Could another E headword replace this one while leaving the invariant or domain line true? If yes, revise it.

Proof rule:
Proof remains local. Cross-domain transfer preserves only the declared invariant relation and enters the target domain as [HYP] until locally tested.

E001 each

Common meaning:
Every one of two or more things considered separately.

Example:
Each child received a book.

RRGM bridge:
Each preserves individual count inside a group. It prevents the whole set from erasing the separate identity of each part.

Invariant MI [INV]:
Each preserves separate count for every member considered one at a time.

Active domains:
[FOR], [REL]

Domain entries:
[FOR] Each preserves individual count inside a group. It prevents the whole set from erasing the separate identity of each part.
[REL] Each protects individual count inside a group so collective reference does not erase separate members.

Count condition:
Counts when every member is considered separately rather than only as an undifferentiated group.

Specific transfer boundary:
The counted set and its boundary must be declared; universal-looking language must not silently expand beyond it.
